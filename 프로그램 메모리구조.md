# 프로그램 메모리 구조 #
---
- '코드'   : 컴파일된 소스코드가 기계어 형태로 저장 (읽기 전용)
- 데이터 : 프로그램 시작과 동시에 할당되어 종료까지 유지 ( 전역변수, static 변수)
- 힙     : 사용자가 직접 관리하는 공간, 런타임 중 크기 결정 ( new 등으로 생성된 동적데이터 )
- 스택   : 함수 호출과 관련된 정보를 저장, 자동 관리 ( 지역변수, 매개변수, 복귀 주소)

### 프로그램 LifeCycle ###
#### 1. Loading ####
프로그램 실행 -> 하드디스크의 실행파일이 RAM으로 올라옴
- 코드 영역   : 소스코드가 기계어로 변환되어 저장
- 데이터 영역 : 프로그램 시작과 동시에 전역변수, 정적변수가 자리잡음

#### 2. Executin ####
main이 호출되면서 프로그램이 활동 시작
- Stack : 함수가 호줄될 때마다 함수의 지역변수, 매개변수가 스택에 push. 복귀 시 pop
- Heap  : 개발자가 직접 요청한 메모리를 할당

#### 3. Runtime ####
스택은 높은 주소->낮은 주소로 내려오고 힙은 올라가는 구조
- 스택 한도 초과(너무 깊은 함수 호출, 너무 큰 지역변수 )   -> Stack Overflow
- 힙 한도 초과  (너무 많은 메모리 할당) -> Heap Corruption, Out Of Memory

#### Termination ####
1. main 종료
2. 스택 자동 제거
3. OS가 프로세스 종료
4. OS가 모든 메모리 회수

### 추가 참고 내용 ###
- 값 형식(Value Type) : 변수가 실제 데이터 값을 가짐, {}벗어나면 스택에서 제거됨
  ex) 기본자료형, 구조체, 열거형
- 참조 형식(Reference Type) : 실제 데이터는 힙, 그 주소값은 스택에 저장-> 변수는 메모리의 주소만 가짐
  ex) 클래스, 인터페이스, 배열, 델리게이트 등
